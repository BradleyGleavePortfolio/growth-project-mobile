## Community v3-1 — Challenges (mobile)

Client surface for v3-1 challenges: a feed card, a progress-logging bottom sheet, and a full-screen detail with a **strictly opt-in** cohort leaderboard and encouragement comments. Consumes the v3-1 backend contract.

### Files (owned)
- `src/screens/community/CommunityChallengeDetailScreen.tsx` — route param `challengeId`; leads with the caller's **own** progress, primary action Join → Log progress, opt-in leaderboard, encouragement comments (reportable). Real loading / empty / error states (no spinner-only), reduced-motion via the sheet, flag-gated with a defense-in-depth flag-off body.
- `src/components/community/ChallengeCard.tsx` — own-progress card; one adaptive action (Join / Continue / View); calm "Goal reached" closure with a line check icon (no badge theater).
- `src/components/community/ChallengeProgressSheet.tsx` — monotonic progress input, animated fill, **reduced-motion aware**, positive completion preview.
- `src/api/communityChallengesApi.ts` — typed Zod-validated client mirroring `communityApi.ts` (idempotency keys on mutations; a drifted response throws a typed `contract` error).

### Shared-append (one-line, distinct anchors)
- `src/config/featureFlags.ts` — `communityChallenges` (env `EXPO_PUBLIC_FF_COMMUNITY_CHALLENGES`, **defaults OFF unconditionally**).
- `.env.example` — `EXPO_PUBLIC_FF_COMMUNITY_CHALLENGES=false`.
- `src/navigation/CommunityNavigator.tsx` — `CommunityChallengeDetail` route registered **only** behind the flag (dead code at build time when OFF).
- `src/screens/community/communityNavTypes.ts` — `CommunityChallengeDetail: { challengeId: string }`.

### Behavioral design (DESIGN_INTELLIGENCE Part III)
Personal progress first, never a ranking by default; the leaderboard is requested **only** once the coach enabled it and the caller opted in, with a calm opt-in/opt-out affordance and zero shame/public-failure framing (§3.2, §3.4, §3.7). Tokens only (no raw hex), line Ionicons only (no emoji), ≥48dp targets.
