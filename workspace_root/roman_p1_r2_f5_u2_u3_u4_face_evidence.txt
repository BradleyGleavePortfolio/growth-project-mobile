     2	 * useRomanChat — chat state machine for RomanChatScreen.
     3	 *
     4	 * Owns: open-or-resume the session, load the first page of messages
     5	 * (newest-first, paged per ListMessagesQueryDto), append a user turn, send it,
     6	 * and fold Roman's settled reply (DECLARED DEVIATION: buffered SSE read — see
     7	 * romanApi header) back into the list. All backend calls go through romanApi,
     8	 * which validates every response against the cited contract and maps errors to
     9	 * the typed RomanApiError union.
    10	 *
    11	 * Concurrency / cleanup (FIFTY_FAILURES #31/#32): an `active` ref gates every
    12	 * post-await setState so a state update never lands after unmount, and an
    13	 * in-flight send is guarded by `sendingRef` so a double-tap cannot fire two
    14	 * turns. Failed sends DO NOT clear the draft — the screen preserves it for
    15	 * retry (brief §3).
    16	 */

    47	/**
    48	 * Outcome of a send attempt, so the screen can distinguish the two failure
    49	 * modes the R1 code audit (F5) requires us to keep separate:
    50	 *   - 'sent'        — the turn persisted; clear the composer.
    51	 *   - 'send-failed' — the turn did NOT persist; keep the draft for retry.
    52	 *   - 'noop'        — nothing was sent (empty/duplicate guard).

    54	export type RomanSendOutcome = 'sent' | 'send-failed' | 'noop';
